PART 1 PIF Perfect → Google Sheets Webhook

~30 min total setup No coding required Free (Google Apps Script)

PIF Perfect's Growth plan ($247/mo) supports outbound webhooks. When a rep submits their daily numbers, PIF fires an HTTP POST to a URL you control. You'll set that URL to a Google Apps Script web app that catches the data and writes it into a Google Sheet. The Projections Calculator then reads from that sheet.

⚠️
Growth plan required. Webhooks are not available on the Starter ($97/mo) plan. If you're on Starter, skip to the manual CSV path at the bottom of this section.

What flows automatically

  • Rep name
  • Submission date
  • Calls made
  • Sets booked
  • Live calls
  • Closed deals
  • Revenue generated

    • What gets auto-calculated

  • Close rate % per rep
  • Set rate % per rep
  • Show rate % per rep
  • MTD totals (all reps)
  • MTD averages
  • Per-rep leaderboard

    • Step 1 — Create a new Google Sheet

    1
    Go to sheets.google.com → click "+ New spreadsheet"
    Name it IFCA Sales Data Hub. This sheet will receive all PIF Perfect submissions and be the data source for the Projections Calculator. Share it with Clementine, Kevin, and Erin so they have access.

    Step 2 — Install the Apps Script

    1
    Open the script editor
    Inside your new sheet: Extensions → Apps Script. A new tab opens with a code editor.
    2
    Replace the default code
    Delete everything in the editor. Paste the full contents of the PIF_Webhook_Script.gs file attached to this setup package. The script is ~150 lines and handles all the logic automatically.
    3
    Run the test function first
    In the editor, select testWebhook from the function dropdown and click ▶ Run. It will ask for permissions — approve them. Then check your Google Sheet — you should see a new PIF_Live_Data tab with one row of test data (Janie, 3 closes, $37,500). If that row appears, the script is working.

    Step 3 — Deploy as a Web App

    ℹ️
    This creates a public HTTPS URL that PIF Perfect can POST data to. The URL is long and unguessable — treat it like a password.
    1
    Click "Deploy" → "New deployment"
    In the Apps Script editor, top-right corner.
    2
    Configure the deployment
    Select type: Web app

    Description: PIF Perfect Webhook Receiver
    Execute as: Me (your Google account)
    Who has access: Anyone ← this is required so PIF can POST to it
    3
    Copy the Web App URL
    After clicking Deploy, Google shows you a URL. It looks like this:
    https://script.google.com/macros/s/AKfycbx[long random string]/exec
    Copy this. You'll paste it into PIF Perfect in the next step. Save it somewhere safe — you'll need it again if you redeploy.

    Step 4 — Configure the webhook in PIF Perfect

    1
    Log into app.pifperfect.com
    Go to Settings → Integrations → Webhooks (or similar — PIF Perfect's UI may differ slightly; look for "Integrations" or "Developer" in the sidebar).
    2
    Add a new webhook
    Webhook URL: paste your Google Apps Script URL
    Trigger event: Submission Created (or equivalent — fires when a rep submits their daily numbers)
    Format: JSON
    3
    If PIF doesn't have a Webhooks UI
    Email support@pifperfect.com and say: "We're on the Growth plan and want to set up an outbound webhook to send submission data to a Google Apps Script endpoint. Can you enable this and share the payload format?" They'll either enable it in your account or confirm it's on their roadmap.

    Step 5 — Test the live connection

    1
    Submit a test entry in PIF Perfect
    Have one rep (or yourself) submit a daily entry in PIF Perfect. Within 5–10 seconds, a new row should appear in the PIF_Live_Data tab of your Google Sheet.
    2
    Check the Webhook_Log tab
    The script logs every raw payload received. If something isn't mapping correctly (wrong field names), you'll see the raw JSON there. Share it with whoever updates the script (or paste it into Claude and say "fix the field mapping").
    3
    Check the PIF_Summary tab
    This tab auto-updates with MTD totals every time a submission comes in. This is the tab your Projections Calculator will read from.

    Payload field reference

    The script tries several common field name variants automatically. If PIF uses different names, update these in the writeSubmission() function:

    MetricField names the script checksFallback
    Rep namerep_name, user.name, name"Unknown"

    Closed dealsclosed, deals_closed, metrics.closed0

    PART 2 Meta Ads MCP for Claude

    5–15 min setup No coding required Free (Meta's official MCP is free in beta)

    Meta launched their official MCP server on April 29, 2026. It connects Claude directly to your live Meta Ads account — no API keys, no developer app, no App Review wait. You can pull CPL, ROAS, spend, campaign performance, and creative data by just asking Claude in plain English.

    ✅
    This is officially Meta-sanctioned automation. The old concern about "using AI to manage Meta Ads gets your account flagged" is now retired. This is Meta's own integration path, using the same OAuth as Shopify and Mailchimp.

    You have two good options. Option A is the right call for your setup — it's Meta's own server, free, and Craig (your ads manager) can connect his account directly.

    Option A — Meta's Official MCP (Recommended)

    ~7 min Free during beta Official Meta integration
    1
    Find your Meta Business ID
    Go to business.facebook.com → Business Settings → Business Info. Your Business ID is the number listed at the top. It looks like: 123456789012345
    2
    Open Claude Desktop settings
    You need Claude Desktop (the Mac or Windows app, not claude.ai in a browser). Download it at claude.ai/download if you don't have it.

    Once open: Claude menu → Settings → Developer → Edit Config
    3
    Add the Meta MCP server to your config
    The config file is a JSON file. Add this inside the "mcpServers" block, replacing YOUR_BUSINESS_ID with your actual number: 
    {
  "mcpServers": {
    "meta-ads": {
      "url": "https://mcp.meta.com/ads/YOUR_BUSINESS_ID"
    }
  }
}
    If there are already other servers in the config, just add the "meta-ads" block alongside them — don't replace anything.
    4
    Restart Claude Desktop and authenticate
    Quit and reopen Claude Desktop. On the next message you send, Claude will prompt you to authenticate with Meta via Facebook Business OAuth — the same login flow as any third-party app. Log in with the account that has admin access to your ad account. You'll be redirected back to Claude once done.
    5
    Verify it's connected
    Ask Claude: "Show me my Meta ad accounts." It should return your account names and IDs. If it does, you're live.
    ⚠️
    Craig should connect his account, not yours. Craig is your paid acquisition manager and runs the ad account day-to-day. Have Craig do steps 1–5 on his machine, authenticating with his Meta Business account. That way Claude pulls from the ad account Craig actually manages. You can also connect your account for read-only visibility.

    Option B — Pipeboard (Zero setup, works in browser)

    ~2 min Paid (~$49–99/mo) Works in claude.ai browser (no Desktop required)

    If you want Meta Ads data in Claude without downloading Claude Desktop, Pipeboard is the fastest path. It's a hosted MCP — you connect your Meta account at pipeboard.co, they give you an MCP URL, and you paste it into Claude's MCP settings in the browser.

    1
    Sign up at pipeboard.co and connect your Meta Ads account
    OAuth flow — same as logging into any app with Facebook.
    2
    Copy your MCP URL from Pipeboard dashboard
    It looks like: https://meta-ads.mcp.pipeboard.co/YOUR_TOKEN
    3
    Add to Claude Desktop config or claude.ai settings
    Same JSON config block as Option A, just swap in the Pipeboard URL.

    What to ask Claude once Meta Ads is connected

    These are the specific queries that feed into your Projections Calculator:

    What you needAsk Claude this
    MTD ad spend"What's our total Meta ad spend month-to-date for [account name]?"
    Cost per lead"What's our CPL on the IFCA VSL campaign for the last 30 days?"
    Cost per booked call"What's cost per booked call (lead form submission) this month?"
    ROAS"What's our ROAS on the Client Engine campaign last 7 days vs. prior 7 days?"
    Top performing creatives"Which ad creatives have the lowest CPL this month? Show me top 5."
    Campaign pacing"Are we on pace to hit $88K spend this month? What's the daily run rate?"

    PART 3 Connecting to the Projections Calculator

    Once both integrations are live, here's how the monthly projections meeting workflow changes:

    1
    Before the meeting (5 min — Clementine or Kevin)
    Open the PIF_Summary tab in the Google Sheet. Copy the MTD values into the FE Sales tab of the Projections Calculator — closes, live calls, close rate by rep. Since PIF fires in real-time, this data is always current.

    Future upgrade: embed the Google Sheet directly into the calculator so it auto-reads via fetch. We can build that once the webhook is confirmed working.
    2
    At the start of the meeting — Meta Ads pull (Craig)
    Craig opens Claude Desktop with the Meta MCP connected and runs:

    "Give me last month's final numbers: total spend, CPL, total leads assigned, cost per booked call, ROAS generated, ROAS collected."

    Paste those into the Marketing section of the FE Sales tab. Takes 60 seconds.
    3
    Set projections for next month
    With actuals loaded, toggle the history reference bar (Prior Month / 3-Month Avg / 6-Month Avg) and set this month's targets. The calculator auto-computes the gap to $550K.
    💡
    Full automation path (later): Once the Google Sheet webhook is confirmed working and stable for 2–3 months, we can add a IMPORTRANGE() formula in the calculator that auto-pulls from PIF_Summary, and a Google Apps Script that also calls the Meta Ads API on a nightly schedule. At that point the calculator updates itself. That's a ~2 hour build and worth doing once the data is reliable.
